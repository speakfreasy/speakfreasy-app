## Notes / Known Issues

- **Email** is stored in `auth.users`, NOT in `profiles`. Never query `profiles.email` — it doesn't exist.
- **`profiles.role`** is a plain text column. Always normalize with `.trim().toLowerCase()` before comparing in app code.
- **`user_access`** is a global flag table only (`has_any_active_sub`). For per-hall access checks, use the `subscriptions` table.
- **`comments`** uses `post_id` / `video_id` FK columns — NOT `parent_type` / `parent_id`.
- **`videos.status`** lifecycle: `uploading → processing → published | failed`. RLS `videos_select` gates on `status = 'published'`. Creators/admins bypass via `videos_select_creator_admin`. Do NOT use `'ready'`.
- **`videos` table** has no `author_id` FK — creator ownership is inferred via `halls.creator_id`. Do not join `profiles` from `videos`.
- **`halls.creator_id`** FK → `profiles` (stores `profiles.id` = `auth.uid()`). Active RLS policies (`halls_select`, `halls_update_own`) check `creator_id = auth.uid()` directly and work correctly. `user_owns_hall()` is vestigial — it checks `creators.id = halls.creator_id` which can never match; it is no longer used in any active policy.
- **`SPEAKFREASY` storage bucket** — 50 MB limit, no MIME restriction. Purpose TBD (likely video uploads).
- **Test account:** `dev@speakfreasy.ca` → `profiles.role = 'admin'`
